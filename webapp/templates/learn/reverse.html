Reverse Engineering

Décompiler, analyser et comprendre un programme sans accès au code source

Élevé 📖 20 min ⚡ Intermédiaire

🔍 Qu'est-ce que le Reverse Engineering ?

Le reverse engineering (ou rétro-ingénierie) consiste à analyser un programme sans accès à son code source pour comprendre son fonctionnement interne. C'est comme démonter une montre pour comprendre comment elle tourne — sauf qu'ici, la montre est un exécutable binaire.

En cybersécurité, le reverse engineering est utilisé pour analyser des malwares, trouver des vulnérabilités, contourner des protections ou, dans le cadre des CTF, retrouver un mot de passe ou une clé cachée dans un binaire.

💡
Usages légitimes du reverse engineering :

Analyse de malware, audit de sécurité, récupération de protocoles propriétaires, interopérabilité logicielle, et bien sûr… les CTF !

📄

📦 Les types de binaires que tu rencontreras

Selon le langage et la plateforme, les binaires n'ont pas la même structure ni les mêmes outils de reverse :

🐧 ELF (Linux)

Format d'exécutable Linux. Produit par GCC, Clang, etc. Outils : ghidra, gdb, radare2.

🪟 PE (Windows)

Format .exe / .dll Windows. Outils : x64dbg, IDA Pro, PE-bear.

🐍 PyInstaller

Script Python empaqueté en binaire. Outils : pyinstxtractor, pycdc, uncompyle6.

☕ JAR (Java)

Bytecode JVM. Facilement décompilable. Outils : jadx, procyon.

🎯
Dans ce cours, on se concentre sur PyInstaller

C'est le cas le plus courant dans les CTF débutants/intermédiaires et c'est ce que tu rencontreras dans notre challenge. Le code Python décompilé reste lisible — pas besoin de maîtriser l'assembleur x86.

🛠️ Méthodologie d'un reverse PyInstaller

Voici les 4 étapes à suivre pour reverser un binaire PyInstaller :

1

Analyse statique préliminaire

Avant tout, examine le binaire à froid sans l'exécuter. Tu peux y trouver des chaînes de caractères révélatrices avec strings :

🔍 Analyse statique Bash 
# Affiche toutes les chaînes lisibles dans le binaire
strings ./licenseguard | grep -i "flag\|password\|secret\|key"

# Détermine le type de fichier
file ./licenseguard
# Output: ELF 64-bit LSB executable...

⚠️ Attention aux fausses pistes !

Les développeurs expérimentés placent parfois de fausses chaînes dans leur binaire pour tromper le reverse engineer. Si tu vois un flag trop facilement… méfie-toi.

2

Extraction du bytecode PyInstaller

Un binaire PyInstaller est en réalité un archive ZIP embarquée qui contient l'interpréteur Python + ton script compilé en bytecode .pyc. L'outil pyinstxtractor permet de tout extraire :

🔧 Extraction Bash 
# Télécharge pyinstxtractor
wget https://github.com/extremecoders-re/pyinstxtractor/raw/master/pyinstxtractor.py

# Extrait le contenu du binaire
python3 pyinstxtractor.py licenseguard

# Résultat : dossier licenseguard_extracted/
# Dedans : licenseguard.pyc + toutes les dépendances
Ce que tu obtiens : Un dossier avec tous les fichiers .pyc dont le script principal — celui qui contient la logique de vérification.
3

Décompilation du bytecode .pyc

Un fichier .pyc est du bytecode Python — il n'est pas directement lisible mais peut être décompilé en Python lisible avec les bons outils :

🔧 Décompilation Bash 
# Option 1 : pycdc (recommandé, supporte Python 3.11+)
git clone https://github.com/zrax/pycdc
cd pycdc && cmake . && make
./pycdc ../licenseguard_extracted/licenseguard.pyc

# Option 2 : uncompyle6 (Python <= 3.8)
pip install uncompyle6
uncompyle6 licenseguard_extracted/licenseguard.pyc

# Option 3 : decompile3 (fork actif)
pip install decompile3
pycdc licenseguard_extracted/licenseguard.pyc

Tu obtiens du code Python quasi identique à l'original. La logique de vérification est maintenant visible !

4

Analyse du code et inversion de la logique

C'est là que le vrai travail commence. Tu lis le code décompilé et tu cherches les fonctions de vérification pour les inverser mathématiquement.

🔎 Exemple de code décompilé Python 
def verify_password(password):
    # XOR chaque caractère avec une clé
    key = [0x4C, 0x47, 0x32, 0x34]
    expected = [0x1a, 0x2b, 0x3c, 0x4d]

    for i, char in enumerate(password.encode()):
        if (char ^ key[i]) != expected[i]:
            return False
    return True

Comment inverser un XOR ?

Le XOR est sa propre inverse : si A XOR K = B, alors B XOR K = A.

Donc pour retrouver le mot de passe :

✅ Script d'inversion Python 
key      = [0x4C, 0x47, 0x32, 0x34]
expected = [0x1a, 0x2b, 0x3c, 0x4d]

# On XOR expected avec key pour retrouver le mot de passe
password = ''.join(chr(e ^ k) for e, k in zip(expected, key))
print(password)  # → le mot de passe en clair !

🛡️ Techniques de protection courantes

Les développeurs utilisent diverses techniques pour rendre le reverse plus difficile. Les connaître t'aide à les contourner :

🚫 Anti-debug

Détecte si un debugger est attaché et quitte le programme.

if sys.gettrace() is not None: sys.exit(1)

Contournement : Patch le binaire ou analyse statiquement sans l'exécuter.

🔒 Flag XORé en mémoire

Le flag n'est jamais en clair dans le binaire — il est encodé et décodé à la volée.

encoded = [0x0f, 0x3d, 0x25] key = b"secret" flag = ''.join(chr(b ^ key[i%len(key)]) for i,b in enumerate(encoded))

Contournement : Reproduis le décodage en Python une fois la logique comprise.

🎭 Fausse piste

Une variable FAKE_FLAG visible dans les strings pour décourager l'analyse.

FAKE_FLAG = "CTF{fake_try_harder}" # Le vrai flag est ailleurs...

Contournement : Lis tout le code, ne t'arrête pas au premier flag trouvé.

🧮 Vérifications multiples

Le mot de passe est découpé en blocs vérifiés indépendamment avec des algos différents.

check_bloc1(b1) # XOR check_bloc2(b2) # somme check_bloc3(b3) # rotation bits check_bloc4(b4) # hash SHA256

Contournement : Inverse chaque vérification dans l'ordre, bloc par bloc.

🧮 Les opérations bitwise à connaître

Le reverse engineering implique souvent de comprendre des opérations sur les bits. Voici les plus courantes en CTF :

XOR ^
1 si les bits sont différents, 0 sinon. Sa propre inverse.
0101 ^ 0011 = 0110
💡 Si A ^ K = B, alors B ^ K = A
AND &
1 uniquement si les deux bits sont à 1. Utilisé pour masquer des bits.
1101 & 1010 = 1000
💡 val & 0xFF garde les 8 bits de poids faible
OR |
1 si au moins un des bits est à 1. Utilisé pour combiner des bits.
0101 | 0011 = 0111
💡 Souvent combiné avec & pour les rotations
Rotation >> / <<
Décale les bits à droite ou à gauche. La rotation fait revenir les bits sortants.
11010000 >>3 = 00011010
💡 Rotation droite = inverse de rotation gauche

📝 Exemple complet : reverser une vérification de mot de passe

Voici un exemple concret de bout en bout. Imagine que tu as extrait ce code décompilé :

🔎 Code décompilé trouvé Python 
import hashlib

_KEY  = [0x41, 0x42, 0x43, 0x44]
_SUM  = 394
_HASH = "5d41402abc4b2a76b9719d911017c592"

def verify(password):
    data = password.encode()

    # Vérification 1 : XOR
    xored = [b ^ _KEY[i % 4] for i, b in enumerate(data[:4])]
    if xored != [0x21, 0x63, 0x26, 0x25]:
        return False

    # Vérification 2 : Somme
    if sum(data[4:8]) != _SUM:
        return False

    # Vérification 3 : Hash MD5
    if hashlib.md5(data[8:]).hexdigest() != _HASH:
        return False

    return True

Pour inverser chaque vérification :

✅ Script d'inversion complet Python 
import hashlib

# ── Bloc 1 : inverser le XOR ──
_KEY    = [0x41, 0x42, 0x43, 0x44]
_xored  = [0x21, 0x63, 0x26, 0x25]
bloc1   = ''.join(chr(x ^ k) for x, k in zip(_xored, _KEY))
print(f"Bloc 1 : {bloc1}")  # → "part"

# ── Bloc 2 : 4 octets dont la somme = 394 ──
# On cherche 4 caractères imprimables qui somment à 394
# Ex: 'a'(97) + 'a'(97) + 'a'(97) + 'm'(109) = 400... on tâtonne
# Ou on cherche intelligemment :
for c1 in range(32, 127):
    for c2 in range(32, 127):
        for c3 in range(32, 127):
            c4 = 394 - c1 - c2 - c3
            if 32 <= c4 < 127:
                bloc2 = chr(c1)+chr(c2)+chr(c3)+chr(c4)
                print(f"Bloc 2 possible : {bloc2}")
                break  # On prend le premier

# ── Bloc 3 : hash MD5 connu → rainbow table ──
# hashlib.md5(b"hello").hexdigest() == "5d41402abc4b2a76b9719d911017c592"
bloc3 = "hello"
print(f"Bloc 3 : {bloc3}")

print(f"Mot de passe complet : {bloc1 + bloc2 + bloc3}")

📝 Résumé — Réflexes CTF Reverse

✅ Commence toujours par file et strings avant tout
✅ Identifie le type de binaire (ELF natif, PyInstaller, JAR…) pour choisir les bons outils
✅ Ne fais pas confiance au premier flag visible — cherche les fausses pistes
✅ Lis toutes les fonctions de vérification avant de commencer à inverser
✅ Inverse les vérifications dans l'ordre : bloc 1, puis 2, puis 3…
✅ Pour les XOR : A ^ K = B → B ^ K = A. La rotation droite inverse la rotation gauche.
✅ Vérifie ton résultat avec un script Python avant de le soumettre
✅ Utilise des rainbow tables ou CrackStation pour inverser les hash MD5/SHA1
