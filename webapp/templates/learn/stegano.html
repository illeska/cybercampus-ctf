Stéganographie

L'art de cacher un message là où personne ne regarde

Élevé 📖 20 min ⚡ Intermédiaire

🔍 Qu'est-ce que la stéganographie ?

La stéganographie est l'art de dissimuler un message à l'intérieur d'un autre fichier — image, son, vidéo ou texte — de façon à ce que personne ne soupçonne même l'existence du message secret.

Elle se distingue fondamentalement de la cryptographie : là où la crypto rend un message illisible, la stéganographie le rend invisible. Un message chiffré attire l'attention ; un message caché dans une photo de vacances ne la retient pas.

💡
Étymologie :

Du grec steganos (couvert, caché) et graphein (écrire). Littéralement : « écriture couverte ». La pratique remonte à l'Antiquité — Hérodote raconte qu'un espion grec tatoua un message sur le crâne rasé d'un esclave, puis attendit que ses cheveux repoussent avant de l'envoyer.

Stéganographie vs Cryptographie

🔐 Cryptographie Message visible mais illisible : k7$xP#2mQz9@Lw...

L'ennemi sait qu'un message existe

≠
🖼️ Stéganographie Message invisible dans une image : [photo anodine de paysage]

L'ennemi ne soupçonne rien

En pratique, les deux techniques sont souvent combinées : on cache un message chiffré dans une image. Même si quelqu'un découvre qu'un message est caché, il ne peut pas le lire.

🛠️ Les principales techniques

🖼️ LSB (Least Significant Bit)

Technique la plus courante. Modifie le bit de poids faible de chaque pixel — imperceptible à l'œil.

📐 Grille mathématique

Cache les données dans des pixels à des coordonnées calculées par une formule secrète.

🎵 Audio stéganographie

Insère des données dans les fréquences inaudibles ou les bits de poids faible d'un fichier audio.

📄 Stéganographie textuelle

Cache un message dans les espaces, la ponctuation ou les premières lettres de chaque mot.

🎨 Canaux de couleur

Exploite les canaux R, G, B ou Alpha d'une image pour stocker des données dans l'un d'eux.

📦 Métadonnées EXIF

Dissimule des informations dans les métadonnées d'un fichier image (champs author, comment…).

⚙️ La technique LSB en détail

LSB (Least Significant Bit) est la technique de stéganographie d'image la plus répandue. Elle repose sur une propriété des pixels : chaque canal de couleur (Rouge, Vert, Bleu) est codé sur 8 bits, soit une valeur entre 0 et 255.

Modifier le dernier bit d'un canal change la valeur d'au maximum 1 — ce qui est totalement imperceptible pour l'œil humain.

Principe bit par bit

📌 Exemple visuel Binaire 
# Pixel original (R=200, G=150, B=100)
R: 11001000   G: 10010110   B: 01100100

# On veut cacher le bit "1" dans le canal R
# On remplace le dernier bit par 1
R: 11001001   ← bit modifié (200 → 201, différence invisible)

# On veut cacher le bit "0" dans le canal G
G: 10010110   ← pas de changement (déjà 0)

# On peut stocker 1 bit par canal, soit 3 bits par pixel
# Une image 800x600 = 480 000 pixels → 180 000 octets cachables

Implémentation Python

🔴 Encoder un message (LSB) Python / Pillow 
from PIL import Image

def encode_lsb(image_path, message, output_path):
    img = Image.open(image_path)
    pixels = img.load()
    W, H = img.size

    # Convertir le message en bits
    bits = ''.join(format(ord(c), '08b') for c in message)
    bits += '00000000'  # Marqueur de fin (octet nul)

    idx = 0
    for y in range(H):
        for x in range(W):
            if idx >= len(bits):
                break
            r, g, b = pixels[x, y][:3]

            # Modifier le LSB du canal Rouge
            r = (r & 0b11111110) | int(bits[idx])
            idx += 1

            pixels[x, y] = (r, g, b)

    img.save(output_path)
    print(f"✅ Message encodé dans {output_path}")
🟢 Décoder un message (LSB) Python / Pillow 
from PIL import Image

def decode_lsb(image_path):
    img = Image.open(image_path)
    pixels = img.load()
    W, H = img.size

    bits = ''
    for y in range(H):
        for x in range(W):
            r = pixels[x, y][0]
            bits += str(r & 1)  # Extraire le LSB du canal Rouge

    # Reconvertir les bits en caractères
    message = ''
    for i in range(0, len(bits), 8):
        byte = bits[i:i+8]
        if byte == '00000000':  # Fin du message
            break
        message += chr(int(byte, 2))

    return message

📐 La technique par grille mathématique

Au lieu de parcourir les pixels séquentiellement, on utilise une formule mathématique pour calculer les coordonnées des pixels porteurs de données. Seul celui qui connaît la formule peut extraire le message.

Exemple de formule :

Le i-ème caractère est stocké dans le pixel de coordonnées :
x = (i × 37) mod largeur
y = (i × 53) mod hauteur

Les nombres 37 et 53 sont des clés secrètes. Sans elles, il faudrait tester toutes les combinaisons possibles — ce qui est computationnellement prohibitif.

📌 Extraction par grille Python 
from PIL import Image

def decode_grid(image_path):
    img = Image.open(image_path)
    pixels = img.load()
    W, H = img.size

    # Pixel index 0 → longueur du message stockée dans le canal R
    length = pixels[(0 * 37) % W, (0 * 53) % H][0]

    message = ''
    for i in range(1, length + 1):
        px = (i * 37) % W
        py = (i * 53) % H
        # Le caractère est stocké dans le canal Rouge (valeur ASCII)
        message += chr(pixels[px, py][0])

    return message
🔑
Pourquoi cette technique est plus robuste que LSB ?

Les outils automatiques de détection LSB (comme StegSolve ou zsteg) ne détecteront rien car les pixels modifiés sont éparpillés de façon pseudo-aléatoire dans l'image. Un analyste doit connaître la formule pour savoir où chercher.

🔬 Détecter la stéganographie

L'analyse stéganalytique est la discipline inverse : détecter qu'une image contient un message caché, voire l'extraire. Voici les principales méthodes :

1

Analyse visuelle des canaux

Isoler chaque canal de couleur (R, G, B) et amplifier les variations permet parfois de visualiser les zones modifiées. L'outil StegSolve est conçu pour ça.

🔍 Avec StegSolve Bash 
# Télécharger et lancer StegSolve
java -jar StegSolve.jar

# Naviguer entre les plans de bits (Bit Plane 0 = LSB)
2

Analyse statistique (histogramme)

Un fichier image normal a une distribution statistique régulière des valeurs de pixels. L'insertion de données LSB introduit des anomalies détectables — notamment une sur-représentation des valeurs paires ou impaires selon le message caché.

3

Outils automatiques

  • zsteg — détection LSB dans les PNG/BMP
  • steghide — cache et extrait des données (JPEG, BMP, WAV)
  • exiftool — analyse les métadonnées EXIF
  • strings — cherche des chaînes lisibles dans un fichier binaire
  • binwalk — détecte des fichiers cachés dans d'autres fichiers
🛠️ Commandes utiles Bash 
# Analyser les métadonnées
exiftool image.png

# Chercher du texte caché dans les bits
zsteg image.png

# Extraire un fichier caché avec steghide
steghide extract -sf image.jpg

# Chercher des signatures de fichiers cachés
binwalk image.png

# Chercher des strings lisibles
strings image.png | grep -i ctf
4

Comparaison avec l'original

Si vous avez accès à l'image originale (avant insertion), une simple différence pixel par pixel révèle exactement quels pixels ont été modifiés.

🔍 Différence d'images Python 
from PIL import Image, ImageChops

original  = Image.open("original.png")
suspecte  = Image.open("suspecte.png")

diff = ImageChops.difference(original, suspecte)
diff.save("difference.png")
# Les pixels modifiés apparaissent en blanc sur fond noir

🌍 Usages réels de la stéganographie

🕵️ Renseignement

Les services secrets utilisent la stéganographie pour transmettre des ordres dissimulés dans des images anodines publiées en ligne.

🦠 Malwares

Des malwares récents (Sunburst, etc.) cachent leurs commandes C2 dans des images PNG pour échapper aux antivirus.

💰 Watermarking

Les studios hollywoodiens cachent un identifiant unique dans chaque copie distribuée pour tracer les fuites.

🔒 Whistleblowers

Des lanceurs d'alerte utilisent la stéganographie pour exfiltrer des documents sans déclencher les systèmes DLP.

⚠️
Dans les CTF :

La stéganographie est un classique des CTF. Les défis vont du simple strings sur un fichier jusqu'à des techniques avancées combinant chiffrement, grilles mathématiques et encodage custom. Ayez toujours exiftool, zsteg, steghide et StegSolve dans votre arsenal.

📝 Résumé des bonnes pratiques CTF

✅ Commencez toujours par exiftool pour analyser les métadonnées
✅ Lancez strings pour chercher du texte lisible caché dans le binaire
✅ Utilisez zsteg pour détecter automatiquement les techniques LSB courantes
✅ Ouvrez l'image dans StegSolve et parcourez les plans de bits
✅ Essayez binwalk — il peut y avoir un fichier ZIP ou PNG caché dans l'image
✅ Regardez les canaux R, G, B séparément — le message peut être dans un seul canal
✅ Si vous trouvez un message, pensez à tester ROT13, Base64, ou d'autres encodages
✅ Lisez l'énoncé attentivement — il contient souvent des indices sur la méthode utilisée
