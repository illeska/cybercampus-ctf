Attaques par Force Brute (Bruteforce)

Comprendre et se protéger contre les attaques par essais successifs

Élevé 📖 18 min ⚡ Débutant

🔍 Qu'est-ce qu'une attaque par force brute ?

Une attaque par force brute (ou bruteforce) est une méthode simple mais efficace qui consiste à tester systématiquement toutes les combinaisons possibles jusqu'à trouver la bonne. C'est comme essayer toutes les clés d'un trousseau jusqu'à trouver celle qui ouvre la serrure.

Contrairement aux attaques par injection (SQLi, XSS) qui exploitent des failles de programmation, le bruteforce exploite simplement le manque de protection contre les tentatives répétées. Si un système ne limite pas le nombre d'essais, un attaquant peut tester des milliers, voire des millions de combinaisons.

💡
Exemple concret :

Imaginez un cadenas à 4 chiffres (0000 à 9999). Il existe 10 000 combinaisons possibles. En testant chaque combinaison une par une, vous êtes CERTAIN de l'ouvrir en maximum 10 000 essais. C'est le principe du bruteforce : la patience et la persévérance remplacent l'intelligence.

🎯 Les différents types de bruteforce

🔤 Bruteforce Simple

Teste toutes les combinaisons possibles de caractères, lettre par lettre.

a, b, c, ... aa, ab, ac, ... aaa, aab, ...

⏱️ Très lent mais garantit de trouver le mot de passe

📖 Attaque par Dictionnaire

Teste une liste de mots de passe courants (dictionnaire).

password, 123456, admin, qwerty, ...

⚡ Très rapide si le mot de passe est commun

🎭 Attaque Hybride

Combine dictionnaire + variations (ajout de chiffres, majuscules, symboles).

password → Password1, p@ssword, password123

⚖️ Équilibré entre vitesse et efficacité

📊
Statistiques impressionnantes :

Avec une machine moderne, on peut tester environ 10 milliards de combinaisons par seconde pour des hashes MD5. Cela signifie qu'un mot de passe de 6 caractères alphanumériques (62^6 = 56 milliards de combinaisons) peut être cracké en moins de 6 secondes !

⚙️ Comment fonctionne une attaque par dictionnaire ?

Prenons l'exemple d'un formulaire de connexion non protégé :

❌ Code vulnérable Python / Flask 
@app.route('/login', methods=['POST'])
def login():
    username = request.form['username']
    password = request.form['password']
    
    # Vérification simple sans limitation
    if username == 'admin' and password == 'secret123':
        return "Connexion réussie !"
    else:
        return "Identifiants incorrects"

Ce code présente plusieurs faiblesses critiques :

Pourquoi ce code est vulnérable ?

  • Aucune limitation du nombre de tentatives - On peut essayer infiniment
  • Pas de délai entre les tentatives - On peut envoyer des milliers de requêtes par seconde
  • Pas de blocage temporaire - Le compte n'est jamais verrouillé
  • Réponse instantanée - Aucun ralentissement artificiel
  • Pas de CAPTCHA - Un script automatisé peut fonctionner sans interruption

Scénario d'attaque avec un script Python :

🔴 Script d'attaque Python 
import requests

# Liste de mots de passe courants
passwords = ['123456', 'password', 'admin', 'qwerty', 'secret123', '12345']

url = 'http://site-vulnerable.com/login'

for pwd in passwords:
    data = {'username': 'admin', 'password': pwd}
    response = requests.post(url, data=data)
    
    if "Connexion réussie" in response.text:
        print(f"✅ Mot de passe trouvé : {pwd}")
        break
    else:
        print(f"❌ Échec avec : {pwd}")
Résultat de l'attaque : ❌ Échec avec : 123456 ❌ Échec avec : password ❌ Échec avec : admin ❌ Échec avec : qwerty ✅ Mot de passe trouvé : secret123

En quelques secondes, l'attaquant a testé 5 mots de passe et trouvé le bon. Avec une liste de 10 000 mots de passe, cela prendrait quelques minutes seulement.

🛡️ Comment se protéger efficacement ?

La protection contre le bruteforce repose sur plusieurs couches de défense :

1

Limitation du nombre de tentatives (Rate Limiting)

Limitez le nombre d'essais autorisés dans une période donnée.

✅ Code sécurisé Python / Flask 
from flask_limiter import Limiter
from flask_limiter.util import get_remote_address

limiter = Limiter(
    app,
    key_func=get_remote_address,
    default_limits=["200 per day", "50 per hour"]
)

@app.route('/login', methods=['POST'])
@limiter.limit("5 per minute")  # Max 5 tentatives par minute
def login():
    # ... code de connexion ...
Résultat : Après 5 tentatives échouées en une minute, l'attaquant doit attendre. Avec cette limite, tester 10 000 mots de passe prendrait plus de 33 heures au lieu de quelques minutes !
2

Délai progressif (Progressive Delay)

Augmentez le temps d'attente après chaque échec successif.

✅ Code sécurisé Python 
import time

failed_attempts = {}  # {ip: nombre_d_echecs}

@app.route('/login', methods=['POST'])
def login():
    ip = request.remote_addr
    
    # Récupérer le nombre d'échecs pour cette IP
    attempts = failed_attempts.get(ip, 0)
    
    # Délai progressif : 2^n secondes (1s, 2s, 4s, 8s, ...)
    if attempts > 0:
        delay = min(2 ** attempts, 60)  # Max 60 secondes
        time.sleep(delay)
    
    # Vérification des identifiants
    if verify_credentials(username, password):
        failed_attempts[ip] = 0  # Reset en cas de succès
        return "Connexion réussie"
    else:
        failed_attempts[ip] = attempts + 1
        return "Identifiants incorrects"
Sans délai progressif : 1000 essais = 10 secondes
→
Avec délai progressif : 1000 essais = plusieurs heures
3

CAPTCHA après plusieurs échecs

Forcez l'attaquant à prouver qu'il est humain après X tentatives échouées.

✅ Code sécurisé Python / Flask 
from flask import session

@app.route('/login', methods=['POST'])
def login():
    attempts = session.get('failed_login_attempts', 0)
    
    # Après 3 échecs, demander un CAPTCHA
    if attempts >= 3:
        captcha = request.form.get('captcha')
        if not verify_captcha(captcha):
            return "CAPTCHA invalide"
    
    if verify_credentials(username, password):
        session['failed_login_attempts'] = 0
        return "Connexion réussie"
    else:
        session['failed_login_attempts'] = attempts + 1
        return render_template('login.html', show_captcha=True)

Les CAPTCHA modernes (reCAPTCHA v3) fonctionnent de manière invisible et ne perturbent pas les utilisateurs légitimes tout en bloquant efficacement les bots.

4

Verrouillage temporaire du compte

Bloquez complètement le compte pendant X minutes après Y tentatives échouées.

  • Exemple : 5 échecs = compte verrouillé pendant 15 minutes
  • Envoyez un email à l'utilisateur légitime pour le prévenir
  • Proposez une réinitialisation de mot de passe par email
  • Loggez toutes les tentatives suspectes pour analyse
⚠️ Attention : Cette méthode peut être exploitée pour faire du déni de service (DoS). Un attaquant pourrait intentionnellement verrouiller des comptes légitimes. Combinez toujours avec d'autres protections.
5

Politique de mots de passe robustes

Imposez des mots de passe difficiles à deviner :

✅ Minimum 12 caractères (idéalement 16+)
✅ Mélange obligatoire : majuscules, minuscules, chiffres, symboles
✅ Interdire les mots de passe communs (liste noire)
✅ Vérifier contre les bases de données de mots de passe compromis (Have I Been Pwned)
✅ Forcer le changement périodique (tous les 6 mois)
✅ Empêcher la réutilisation des 5 derniers mots de passe
Temps pour cracker selon la complexité : • "password" (8 lettres) → Instantané • "Password123" (12 caractères) → Quelques heures • "P@ssw0rd!2024" (13 caractères mixtes) → Plusieurs semaines • "Tr0p!c@l_P@r@d!se_2024" (24 caractères) → Des millions d'années
6

Authentification à deux facteurs (2FA)

Ajoutez une couche de sécurité supplémentaire au-delà du mot de passe :

  • SMS/Email : Code temporaire envoyé sur le téléphone ou email
  • Application d'authentification : Google Authenticator, Authy, etc.
  • Clé de sécurité physique : YubiKey, Titan Security Key
  • Biométrie : Empreinte digitale, reconnaissance faciale
Impact : Même si l'attaquant trouve le mot de passe par bruteforce, il ne pourra pas se connecter sans le second facteur d'authentification. Le 2FA réduit le risque de compromission de 99,9%.

🔬 Détecter une attaque en cours

Comment savoir si votre système est victime d'un bruteforce ?

🚨
Indicateurs d'une attaque :
  • Nombre anormal de requêtes de connexion sur une courte période
  • Multiples échecs de connexion depuis la même IP
  • Tentatives de connexion sur de nombreux comptes différents
  • Pics de trafic inhabituels sur la page de login
  • User-Agent suspects ou absents dans les logs

Outils de monitoring recommandés :

📊 Fail2Ban

Bannit automatiquement les IP suspectes après X échecs

🛡️ Cloudflare

Protection DDoS et rate limiting au niveau réseau

📈 ELK Stack

Analyse en temps réel des logs d'authentification

📝 Résumé des bonnes pratiques

✅ Limitez strictement le nombre de tentatives (rate limiting)
✅ Implémentez un délai progressif après chaque échec
✅ Ajoutez un CAPTCHA après 3-5 tentatives échouées
✅ Verrouillez temporairement les comptes après trop d'échecs
✅ Imposez des mots de passe robustes (12+ caractères)
✅ Activez l'authentification à deux facteurs (2FA)
✅ Surveillez les logs et détectez les patterns suspects
✅ Ne révélez jamais si le problème vient du username ou du password
✅ Hashage sécurisé avec bcrypt/Argon2 (jamais MD5/SHA1)
